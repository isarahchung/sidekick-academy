The mental model

A routine is a structured prompt with branches.

  • Steps run top-to-bottom unless a conditional says otherwise
  • Each step can call tools, ask the model, or set a variable
  • Branches let you handle different user intents in one routine
  • Guardrails sit on top and can short-circuit any step
Distyl 03
Two modes

Chat vs. Simulation.

Chat

Single conversation. You type, the routine runs, you see what happens.

  • Fast iteration on prompts
  • Manual exploration
  • No graded outcome

Simulation

Runs an eval set against a branch. Each example uses a scenario for tool stubs.

  • Deterministic, replayable
  • Graded against HLOs
  • Proves fixes hold

What the agent sees

Anatomy of a routine.

◀ Stable Volatile ▶
01

Context

system + triage

Persona, capabilities, and how to route intents. Same on every interaction.
02

Routine

steps + conditionals

The active flow. Triage picks one per intent; steps run top-to-bottom unless a conditional says otherwise.
03

Tools

schemas + responses

Actions the routine can take. Schemas are stable; responses change every call.
04

Conversation

turns so far

User and assistant turns. Appended every exchange; drives the next step.
